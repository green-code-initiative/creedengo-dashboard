Empty methods will end up consuming CPU and RAM although they do nothing. Best is to get rid of empty functions, throw a relevant exception, or at least explain why function is empty in a comment.

Example

Avoid:

        public void doSomething()
        {
        }

Prefer:

        public void doSomething()
        { 
            throw new UnsupportedOperationException();
        }

        public void doSomething()
        { 
            // At least explain why nothing is happening yet
        }
