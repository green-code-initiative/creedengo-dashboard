By definition less specific, generic exceptions are usually handled in a less precise way, which can lead to the wrong level of processing. They might therefore consume more CPU or RAM than necessary.

Example

Avoid:

        public void foo(String bar) throws Exception
        {
            throw new Exception("My Message");    
        }

Prefer:

        public void foo(String bar) throws MyCustomException
        {
            throw new MyCustomException("My Message");
        }
